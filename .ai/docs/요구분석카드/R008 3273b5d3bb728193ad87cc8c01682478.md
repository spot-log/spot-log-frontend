# R008

사용자스토리: 사용자가 공개 메모 작성 시 유효 기간(1일/1주/1개월/3개월/직접 입력)을 설정할 수 있다 (https://www.notion.so/1-1-1-3-3223b5d3bb7281e985c4fd4b8b7b548b?pvs=21)
업무영역: 백엔드, 프론트엔드
유형: 기능

## 요약

공개 메모 작성 시 유효 기간 설정 기능. 사전 정의된 기간 옵션(1일/1주/1개월/3개월) 또는 직접 입력으로 만료일을 지정함.

## 수용 기준 (Acceptance Criteria)

- [ ]  공개 메모 작성 폼에 유효 기간 선택 UI가 존재함
- [ ]  1일 / 1주 / 1개월 / 3개월 / 직접 입력 옵션 제공
- [ ]  직접 입력 선택 시 날짜 선택기(date picker) 활성화
- [ ]  유효 기간 미선택 시 저장 불가(필수값 처리)
- [ ]  저장된 메모의 만료일이 DB에 정확히 기록됨
- [ ]  개인 메모는 유효 기간 설정 UI가 노출되지 않음

## 작업 단계 (Implementation Steps)

### 프론트엔드

1. 메모 유형이 '공개'로 선택된 경우에만 유효 기간 선택 컴포넌트 렌더링
2. 라디오 버튼 또는 세그먼트 컨트롤로 사전 정의 기간 옵션 구현
3. '직접 입력' 선택 시 날짜 선택기 표시 및 값 바인딩
4. 폼 제출 시 선택 기간 → 만료 일시(ISO 8601) 변환 후 API 요청

### 백엔드

1. 메모 생성 요청 바디에 `expiredAt` 필드(ISO 8601) 수신
2. 공개 메모(`type=PUBLIC`)인 경우 `expiredAt` 필수값 검증 — 미포함 시 400 반환
3. `expiredAt`이 현재 시각 이전인 경우 400 반환
4. 검증 통과 시 메모 엔티티에 만료일 저장

## 제약 사항 (Constraints)

- 개인 메모(`type=PRIVATE`)는 `expiredAt` 불필요 — 서버에서도 무시 처리
- 직접 입력 최대 허용 기간은 별도 정책 결정 전까지 제한 없음
