# R022

사용자스토리: 사용자가 동일 메모에 대해 반경 이탈 후 재진입 시에만 재알림을 받는다 (https://www.notion.so/3223b5d3bb72811e915df2d036200b1b?pvs=21)
업무영역: 프론트엔드
유형: 기능

## 요약

지오펜싱 알림 중복 방지 기능. 동일 메모에 대해 반경 이탈 후 재진입 시에만 재알림이 발송됨.

## 수용 기준 (Acceptance Criteria)

- [ ]  반경 진입 후 이탈 없이 이동 중일 때 중복 알림 미발송
- [ ]  반경 이탈 후 재진입 시 알림 재발송
- [ ]  앱 재시작 후에도 진입 상태가 유지되거나 최초 진입으로 처리됨

## 작업 단계 (Implementation Steps)

### 프론트엔드

1. 메모별 진입 상태 플래그를 로컬 상태(또는 영구 저장소)로 관리
2. 위치 변경 이벤트마다 각 메모의 진입 여부 계산
3. 이전 상태가 '이탈' → 현재 상태가 '진입'인 경우에만 알림 발송
4. '진입' 상태에서 이탈 감지 시 플래그를 '이탈'로 갱신

## 제약 사항 (Constraints)

- 진입/이탈 상태는 메모 ID별로 독립 관리
- 앱 재시작 후 상태 초기화 시 최초 진입으로 간주하여 알림 발송 허용
