# R009

사용자스토리: 사용자가 만료된 메모를 삭제하거나 재공개할 수 있다 (https://www.notion.so/3223b5d3bb72811b8389e1057a9752f1?pvs=21)
업무영역: 백엔드, 프론트엔드
유형: 기능

## 요약

만료된 공개 메모에 대한 사용자 처리 기능. 삭제 또는 재공개(유효 기간 재설정) 중 선택 가능함.

## 수용 기준 (Acceptance Criteria)

- [ ]  '만료됨' 탭에서 만료된 본인 메모 목록이 조회됨
- [ ]  각 메모에 '삭제' 및 '재공개' 버튼 존재
- [ ]  '재공개' 선택 시 유효 기간 재설정 모달 표시
- [ ]  재공개 후 해당 메모가 공개 메모 목록 및 지도에 재노출됨
- [ ]  삭제 시 확인 다이얼로그 표시 후 즉시 제거됨

## 작업 단계 (Implementation Steps)

### 프론트엔드

1. '만료됨' 탭 진입 시 만료 메모 목록 API 호출
2. 각 아이템에 삭제/재공개 액션 버튼 렌더링
3. 재공개 버튼 탭 → 유효 기간 선택 모달 표시(R008 컴포넌트 재사용)
4. 재공개 API 호출 성공 시 해당 아이템을 만료됨 탭에서 제거
5. 삭제 확인 다이얼로그 구현 및 삭제 API 호출

### 백엔드

1. 만료 메모 재공개 API(`PATCH /memos/{id}/republish`) 구현
2. 요청자가 메모 소유자인지 검증 — 불일치 시 403 반환
3. 새 `expiredAt` 값 수신 및 유효성 검증(현재 시각 이후)
4. 메모 만료일 갱신 및 상태를 활성 공개 메모로 복원
5. 삭제 API는 기존 R006 스펙 준용

## 제약 사항 (Constraints)

- 재공개는 본인 소유 메모에 한정
- 타인의 만료된 공개 메모는 재공개 불가
