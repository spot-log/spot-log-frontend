# R014

사용자스토리: 사용자가 본인이 작성한 메모를 삭제할 수 있다 (삭제 확인 다이얼로그 포함) (https://www.notion.so/3223b5d3bb728123b6d4def5d248baa8?pvs=21)
스프린트 배정: 1차
업무영역: 백엔드, 프론트엔드
유형: 기능

## 요약

본인이 작성한 메모를 삭제하는 기능. 삭제 확인 다이얼로그를 통해 오조작 방지.

## 수용 기준 (Acceptance Criteria)

- [ ]  메모 상세 또는 목록에서 삭제 버튼 접근 가능
- [ ]  삭제 버튼 탭 시 확인 다이얼로그 표시 ('삭제하시겠습니까?' + 확인/취소)
- [ ]  확인 시 메모 즉시 삭제 및 목록에서 제거
- [ ]  취소 시 다이얼로그 닫힘, 메모 유지
- [ ]  타인 메모에 대한 삭제 버튼 미노출

## 작업 단계 (Implementation Steps)

### 프론트엔드

1. 메모 소유자 여부 확인 후 삭제 버튼 조건부 렌더링
2. 삭제 버튼 탭 → 확인 다이얼로그 표시
3. 확인 → DELETE API 호출 → 성공 시 목록 갱신
4. API 오류 시 에러 토스트 메시지 표시

### 백엔드

1. `DELETE /memos/{id}` API 구현
2. 인증 토큰에서 사용자 ID 추출 및 메모 소유자 검증
3. 소유자 불일치 시 403 반환
4. 존재하지 않는 메모 요청 시 404 반환
5. 검증 통과 시 즉시 삭제 처리

## 제약 사항 (Constraints)

- 삭제는 소프트 딜리트 또는 하드 딜리트 중 팀 정책에 따름
- 삭제된 메모에 연결된 북마크는 자동 제거됨
