# R005

사용자스토리: 사용자가 Google 소셜 로그인으로 회원가입 및 로그인할 수 있다 (https://www.notion.so/Google-3223b5d3bb7281268373f6eaefbbb388?pvs=21)
스프린트 배정: 1차
업무영역: 백엔드, 프론트엔드
유형: 기능

## 요약

Google OAuth 2.0을 통한 소셜 로그인 기능. 신규 사용자는 자동 계정 생성, 기존 사용자는 JWT 발급으로 인증 처리함.

## 수용 기준 (Acceptance Criteria)

- [ ]  로그인 화면에 Google 로그인 버튼 존재
- [ ]  버튼 탭 시 Google OAuth 인증 페이지로 이동
- [ ]  인증 완료 후 앱으로 복귀 및 홈 화면 진입
- [ ]  로그인 실패 시 오류 메시지 표시 ("로그인에 실패했습니다. 다시 시도해주세요")
- [ ]  신규 사용자의 경우 계정 자동 생성 후 JWT 발급
- [ ]  기존 사용자의 경우 AccessToken + RefreshToken 발급

## 작업 단계 (Implementation Steps)

### 프론트엔드

1. 로그인 화면 UI 구현 — Google 로그인 버튼 배치
2. 버튼 탭 이벤트 → OAuth 제공자 인증 페이지로 이동
3. 인증 완료 후 인증 코드(code)를 서버로 전달
4. 성공 응답(JWT) 수신 후 로컬 저장 및 홈 화면 이동
5. 실패 응답 시 오류 메시지 토스트 표시

### 백엔드

1. 클라이언트로부터 Google 인증 코드 수신
2. Google API로 액세스 토큰 교환 및 사용자 정보(이름, 이메일) 획득
    1. 정보 획득 실패 또는 유효하지 않은 데이터 시 400 반환
3. 이메일 기준으로 기존 계정 존재 여부 확인
4. 기존 계정: 해당 계정의 JWT(AccessToken + RefreshToken) 발급
5. 신규 계정: 이름·이메일로 계정 생성 후 JWT 발급

## 제약 사항 (Constraints)

- Google OAuth 2.0 Authorization Code Flow 사용 (Implicit Flow 금지)
- RefreshToken은 HttpOnly 쿠키 또는 보안 저장소에 저장 — 로컬스토리지 저장 금지
- Google API 호출 실패 시 사용자 정보 저장 불가
