# R006

사용자스토리: 사용자가 현재 위치 기반으로 텍스트 메모를 작성하고 지도에 핀으로 저장할 수 있다 (https://www.notion.so/3223b5d3bb72811eb0b9e29cf6e438f7?pvs=21)
스프린트 배정: 1차
업무영역: 백엔드, 프론트엔드
유형: 기능

## 요약

메모 생성·조회·삭제를 통합한 핵심 CRUD 기능. 위치 기반 메모 작성, 개인/공개 유형 분류, 삭제 확인 다이얼로그를 포함함.

## 수용 기준 (Acceptance Criteria)

- [ ]  지도에서 위치 선택 또는 현재 위치 기준으로 메모 작성 폼 진입
- [ ]  제목(선택), 본문(필수), 유형(개인/공개), 반경, 유효기간 입력 가능
- [ ]  공개 메모 작성 시 유효기간 필수 입력
- [ ]  저장 완료 시 해당 좌표에 핀 표시
- [ ]  내 메모 목록에서 개인/공개/만료됨 탭으로 분류 조회 가능
- [ ]  메모 상세 화면에서 삭제 버튼 탭 시 확인 다이얼로그 표시
- [ ]  삭제 확인 시 목록 및 지도에서 즉시 제거

## 작업 단계 (Implementation Steps)

### 프론트엔드

1. 지도에서 위치 선택 또는 현재 위치로 메모 작성 폼 오픈
2. 입력 필드 구현: 제목, 본문, 유형(개인/공개), 반경, 유효기간 (R008 컴포넌트 재사용)
3. 저장 완료 시 핀 렌더링 및 목록 갱신
4. 메모 목록에서 탭(개인/공개/만료됨)별 조회 (R011 컴포넌트 재사용)
5. 삭제 버튼 탭 → 확인 다이얼로그 표시 → 확인 시 DELETE API 호출

### 백엔드

**메모 생성**

1. 인증된 사용자로부터 제목, 본문, 좌표(위도/경도), 반경, 유형(PRIVATE/PUBLIC), 유효기간 수신
2. 본문 1자 미만 또는 1000자 초과 시 400 반환
3. 제목 미입력 시 본문 앞 10자 자동 설정 (R024 준용)
4. 메모 저장 후 생성된 메모 객체 반환

**메모 조회**

1. 내 메모 목록: 인증된 사용자의 메모를 type 파라미터로 필터링 조회 (R011 준용)
2. 주변 메모 조회: 현재 위치 + 탐색 반경 기준 공개 메모 조회 (R015 준용)
3. 만료된 공개 메모는 주변 메모 조회 결과에서 제외

**메모 삭제**

1. 인증된 사용자로부터 메모 ID 수신
2. 본인 소유가 아닌 메모 삭제 요청 시 403 반환
3. 존재하지 않는 메모 삭제 요청 시 404 반환
4. 검증 통과 시 즉시 삭제

## 제약 사항 (Constraints)

- 삭제는 소프트 딜리트 또는 하드 딜리트 중 팀 정책에 따름
- 삭제된 메모에 연결된 북마크는 자동 제거됨
- 공개 메모(`type=PUBLIC`)는 `expiredAt` 필수 — 미포함 시 400 반환
